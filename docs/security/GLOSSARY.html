Security Scanning Glossary

Terms and Definitions

Trivy
A comprehensive, open-source security scanner for containers, file systems, and git repositories. Trivy detects vulnerabilities in OS packages and application dependencies.
SBOM (Software Bill of Materials)
A formal, machine-readable inventory of software components and dependencies, their relationships, and hierarchical structure, used to track and manage security vulnerabilities.
CVE (Common Vulnerabilities and Exposures)
A standardized identifier for known security vulnerabilities, maintained by MITRE Corporation. Each CVE entry includes identification number, description, and references.
SAST (Static Application Security Testing)
A testing methodology that analyzes source code for security vulnerabilities without executing the program. Trivy performs SAST when scanning repositories.
Severity Level
A classification of vulnerabilities based on their potential impact and exploitability. Common levels include CRITICAL, HIGH, MEDIUM, and LOW.
Secret Scanning
The process of detecting credentials, API keys, tokens, and other sensitive information in source code, which should not be committed to repositories.
Dependency Scanning
The process of checking software dependencies (libraries, packages, etc.) for known security vulnerabilities.
Exit Code
A value returned by a process to indicate its completion status. In security scanning, non-zero exit codes typically indicate that vulnerabilities were found.
SARIF (Static Analysis Results Interchange Format)
A JSON-based standard format for the output of static analysis tools, enabling integration with different development and security tools.

Workflow Components

trivy.yaml
Configuration file for Trivy scanner that defines scanning parameters, severity thresholds, and output formats.
trivy-scan.yml
GitHub Actions workflow file that defines the security scanning process, including when scans run, what they scan for, and how results are processed.
workflow_dispatch
A GitHub Actions trigger that allows workflows to be manually executed from the GitHub interface, useful for running on-demand security scans.
Concurrency Control
A mechanism in GitHub Actions that prevents multiple instances of the same workflow from running simultaneously, optimizing resource usage.
Submodule Initialization
The process of initializing and updating Git submodules, which are Git repositories nested within another Git repository, ensuring they are also scanned for vulnerabilities.
