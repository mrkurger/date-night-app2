Terms and Concepts

LSP (Language Server Protocol)
Protocol
A protocol that allows code editors to communicate with language servers, which provide language features like code completion, error checking, and hover information.
MCP (Model Context Protocol)
Protocol
A protocol that allows AI models to interact with external tools and resources, enabling them to access information beyond their training data.
LSP-MCP
Integration
An integration that bridges the Language Server Protocol and the Model Context Protocol, allowing AI models to access language server features.
Language Server
Component
A server that implements the Language Server Protocol to provide language features like code completion, error checking, and hover information for a specific programming language.
TypeScript Language Server
Component
A language server that provides language features for TypeScript and JavaScript, including type checking, code completion, and hover information.
Hover Information
Feature
Information displayed when hovering over a code element, typically including type information, documentation, and other relevant details.
Code Completion
Feature
Suggestions for completing code as it is being typed, based on the context and available symbols.
Diagnostics
Feature
Error messages, warnings, and other information about potential issues in the code.
Code Actions
Feature
Suggested actions to fix issues or improve code, such as adding missing imports or fixing type errors.

Functions and Tools

start_lsp
MCP Tool
Starts the LSP server with a specified root directory. This must be called before using any other LSP-related tools.
{ "tool": "start_lsp", "arguments": { "root_dir": "/path/to/your/project" } }
open_document
MCP Tool
Opens a file in the LSP server for analysis. This must be called before accessing diagnostics or performing other operations on the file.
{ "tool": "open_document", "arguments": { "file_path": "/path/to/your/file.ts", "language_id": "typescript" } }
close_document
MCP Tool
Closes a file in the LSP server when you're done working with it. This helps manage resources and cleanup.
{ "tool": "close_document", "arguments": { "file_path": "/path/to/your/file.ts" } }
get_info_on_location
MCP Tool
Gets hover information at a specific location in a file, including type information, documentation, and other relevant details.
{ "tool": "get_info_on_location", "arguments": { "file_path": "/path/to/your/file.ts", "language_id": "typescript", "line": 3, "column": 5 } }
get_completions
MCP Tool
Gets completion suggestions at a specific location in a file, based on the context and available symbols.
{ "tool": "get_completions", "arguments": { "file_path": "/path/to/your/file.ts", "language_id": "typescript", "line": 3, "column": 10 } }
get_diagnostics
MCP Tool
Gets diagnostic messages (errors, warnings) for one or all open files.
{ "tool": "get_diagnostics", "arguments": { "file_path": "/path/to/your/file.ts" } }
get_code_actions
MCP Tool
Gets code actions for a specific range in a file, such as suggested fixes for errors or code improvements.
{ "tool": "get_code_actions", "arguments": { "file_path": "/path/to/your/file.ts", "language_id": "typescript", "start_line": 3, "start_column": 5, "end_line": 3, "end_column": 10 } }
restart_lsp_server
MCP Tool
Restarts the LSP server process without restarting the MCP server. This is useful for recovering from LSP server issues or for applying changes to the LSP server configuration.
{ "tool": "restart_lsp_server", "arguments": { "root_dir": "/path/to/your/project" } }
set_log_level
MCP Tool
Sets the server's logging level to control verbosity of log messages.
{ "tool": "set_log_level", "arguments": { "level": "debug" } }

Resources

lsp-diagnostics://
MCP Resource
A resource scheme for accessing diagnostic information for open files.
lsp-diagnostics:///path/to/file.ts
lsp-hover://
MCP Resource
A resource scheme for accessing hover information at specific positions in files.
lsp-hover:///path/to/file.ts?line=3&column=5&language_id=typescript
lsp-completions://
MCP Resource
A resource scheme for accessing code completion suggestions at specific positions in files.
lsp-completions:///path/to/file.ts?line=3&column=10&language_id=typescript
