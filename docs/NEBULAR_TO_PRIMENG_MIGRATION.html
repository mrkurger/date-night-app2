Nebular to PrimeNG Migration

1. Overview

This document tracks the migration of components from Nebular to PrimeNG. The migration follows a component-first approach, prioritizing standalone components and commonly used UI elements.

Current Setup

  • Main Nebular configuration in nebular.module.ts and nebular-custom.module.ts
  • Custom Nebular components in shared/components/custom-nebular-components
  • Several standalone components already using PrimeNG successfully

2. Key Components to Migrate

Layout Components

  • ✅ app-card → p-card
  • ⏳ nb-layout → PrimeNG layout
  • ⏳ nb-sidebar → p-sidebar
  • ⏳ nb-menu → p-menubar/p-menu

Form Components

  • ⏳ nb-form-field → p-inputgroup
  • ⏳ nb-input → p-input
  • ⏳ nb-select → p-dropdown

Data Display

  • ⏳ nb-data-table → p-table
  • ⏳ nb-list → p-listbox
  • ⏳ nb-tree-grid → p-treegrid

Feedback Components

  • ⏳ nb-spinner → p-progressSpinner
  • ⏳ nb-alert → p-message
  • ⏳ nb-tooltip → p-tooltip

4. Migration Strategy

Step 1: Replace Core Layout

  • Replace nb-layout with PrimeNG equivalent layout structure
  • Migrate sidebar navigation to p-sidebar
  • Update menu system to use p-menubar/p-menu

Step 2: Migrate Standalone Components

  • Identify and update existing standalone components
  • Convert high-traffic components to standalone + PrimeNG
  • Create new components as standalone with PrimeNG

Step 3: Migrate NgModule-based Components

  • Convert module-based components to standalone
  • Replace Nebular components with PrimeNG equivalents
  • Update templates and styles

Step 4: Clean Up

  • Remove @nebular/* dependencies
  • Clean up SCSS/styles
  • Remove Nebular theme customization
  • Update global styles to PrimeNG theme system

5. Dependencies

Current

  • @nebular/eva-icons
  • @nebular/security

Target

  • @primeng/core
  • primeng (all needed modules)
  • primeicons

6. Common Challenges

  • Theme variable migration from Nebular to PrimeNG
  • Layout structure differences between frameworks
  • Custom component adaptations
  • Event binding syntax changes
  • Icon system migration (eva-icons to PrimeIcons)
