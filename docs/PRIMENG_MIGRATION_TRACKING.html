Migration Notes

Common Issues

  • Nebular theme variables need to be replaced with PrimeNG variables
  • Some components require structural changes in templates
  • Event bindings may need updates
  • Custom components need careful planning for migration

Best Practices

  • Always make components standalone when possible
  • Use PrimeNG's built-in features instead of custom implementations
  • Keep SCSS organized and use PrimeNG variables
  • Document complex migrations in component-specific README files
