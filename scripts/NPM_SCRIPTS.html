Startup Scripts

start
npm run start
Main entry point to start both server and client concurrently. Uses the concurrently package to run both the server and client in parallel.
start:server
npm run start:server
Starts only the server in development mode. Changes directory to the server folder and runs its dev script.
start:client
npm run start:client
Starts only the Angular client application. Uses a custom script (scripts/start-client.js) that handles directory changes and memory allocation.
The script increases Node.js memory allocation to 8GB to prevent memory issues during Angular compilation.
dev
npm run dev
Development startup script that fixes common issues before starting the application. Runs fix-project.sh to fix common issues, then starts both server and client.
This is the recommended script for local development as it ensures the environment is properly set up.

Installation & Setup Scripts

install-all
npm run install-all
Installs dependencies for the root project, server, and client. Runs npm install in each directory sequentially.
install-missing
npm run install-missing
Installs missing dependencies identified by the custom script. Uses scripts/install-missing-deps.js to check and install missing dependencies.
This script checks for common dependencies and also installs specific known missing dependencies.
preinstall
npm run preinstall
Runs automatically before npm install to disable husky in CI environments. Uses scripts/disable-husky-in-ci.js to create a .huskyrc file.
This prevents Git hook issues in CI environments.
prepare
npm run prepare
Sets up husky Git hooks after npm install (but skips in CI environments). Uses a shell command to check for CI environment before running husky.

MongoDB Scripts

mongodb
npm run mongodb
Ensures MongoDB is running for the application. Uses scripts/ensure-mongodb.js to check and start MongoDB if needed.
The script handles different operating systems and error cases.
fix-mongodb
npm run fix-mongodb
Fixes common MongoDB issues. Uses scripts/fix-mongodb-issues.js to repair MongoDB installations.
check-mongodb
npm run check-mongodb
Checks MongoDB permissions. Uses scripts/check-mongodb-permissions.js to verify proper access rights.
setup:mongodb
npm run setup:mongodb
Combines MongoDB permission check and issue fixing. Runs check-mongodb and fix-mongodb scripts in sequence.
This is a convenient helper script for MongoDB setup.

Content Security Policy (CSP) Scripts

fix-csp
npm run fix-csp
Fixes Content Security Policy issues in both server and client. Uses scripts/fix-csp-issues.js to update CSP configurations.
fix-csp:restart
npm run fix-csp:restart
Fixes CSP issues and restarts the application. Runs fix-csp with the --restart flag.

ESM (ECMAScript Modules) Scripts

convert-to-esm
npm run convert-to-esm
Converts CommonJS modules to ESM. Uses scripts/convert-to-esm.js to update import/export syntax.
fix-esm
npm run fix-esm
Fixes remaining ESM issues after conversion. Uses scripts/fix-remaining-esm-issues.js to resolve edge cases.
validate-esm
npm run validate-esm
Validates ESM compatibility. Uses scripts/validate-esm.js to check for ESM compliance.

Maintenance Scripts

make-executable
npm run make-executable
Makes all scripts executable. Uses make-scripts-executable.sh to set executable permissions.
fix-all
npm run fix-all
Runs the comprehensive fix script. Uses fix-project.sh to fix various issues.
check-node
npm run check-node
Checks Node.js version compatibility. Uses scripts/check-node-version.js to verify Node.js version.
update-packages
npm run update-packages
Updates package dependencies. Uses scripts/update-packages.js to update to compatible versions.
audit-fix
npm run audit-fix
Fixes npm audit issues using the standard approach (without --force).
audit-fix:force
npm run audit-fix:force
Fixes npm audit issues using the --force flag. This may introduce breaking changes.
Use with caution as it may update dependencies to versions with breaking changes.
audit-fix:all
npm run audit-fix:all
Runs npm audit fix in the root project, server, and client directories.

Testing Scripts

test
npm run test
Runs server tests. Alias for test:server.
test:server
npm run test:server
Runs all server tests. Changes directory to server and runs its test script.
test:server:unit
npm run test:server:unit
Runs only server unit tests. Uses Jest with a specific test match pattern.
test:server:integration
npm run test:server:integration
Runs only server integration tests. Uses Jest with a specific test match pattern.
test:server:performance
npm run test:server:performance
Runs only server performance tests. Uses Jest with a specific test match pattern.
test:client
npm run test:client
Runs client tests. Changes directory to client-angular and runs its test script.
test:client:coverage
npm run test:client:coverage
Runs client tests with coverage reporting. Changes directory to client-angular and runs its test:coverage script.
test:all
npm run test:all
Runs both server and client tests. Runs test:server and test:client in sequence.
test:watch
npm run test:watch
Runs tests in watch mode. Uses Jest's watch mode to rerun tests on file changes.
test:coverage
npm run test:coverage
Runs tests with coverage reporting. Uses Jest's coverage reporting.
test:ci
npm run test:ci
Runs tests in CI mode. Uses Jest's CI mode with JUnit reporter for CI integration.

Linting & Formatting Scripts

lint
npm run lint
Runs linting for both server and client. Runs lint:server and lint:client in sequence.
This will fail if there are linting errors.
lint:server
npm run lint:server
Runs linting for server code. Changes directory to server and runs its lint script.
This will fail if there are linting errors.
lint:client
npm run lint:client
Runs linting for client code. Changes directory to client-angular and runs its lint script.
This will fail if there are linting errors.
lint:check
npm run lint:check
Checks for linting issues without failing. Runs lint:server:check and lint:client:check in sequence.
This will not fail even if there are linting errors.
lint:fix
npm run lint:fix
Fixes linting issues for both server and client. Runs lint:server:fix and lint:client:fix in sequence.
format
npm run format
Formats code using Prettier. Formats files in the src directory.
format:server
npm run format:server
Formats server code. Changes directory to server and runs its format script.
format:client
npm run format:client
Formats client code. Changes directory to client-angular and runs its format script.
format:all
npm run format:all
Formats all code and fixes linting issues. Runs format and lint:fix in sequence.

Documentation Scripts

docs:improve
npm run docs:improve
Improves documentation. Runs scripts/run_docs_improvement.sh to enhance documentation.
docs:status
npm run docs:status
Shows documentation status. Runs scripts/docs_improvement_helper.js with the --report flag.
docs:rename
npm run docs:rename
Renames documentation files. Runs scripts/rename_docs_to_uppercase.js to standardize filenames.
docs:generate-feature
npm run docs:generate-feature
Generates feature documentation. Runs scripts/generate_feature_docs.js to create feature-specific documentation.
