Glossary: General

This file contains a glossary of terms, tools, and concepts used in the Date Night App project.

Configuration

Prettier is configured using a .prettierrc file at the root of the project. The configuration includes settings for indentation, line length, quotes, and other formatting options.

Usage

// Format all files
npm run format

// Check if files are formatted correctly
npm run format:check

Husky

Tool

A tool that makes it easy to use Git hooks in a Node.js project. Husky is used in the Date Night App project to run code formatting and linting before commits.

Configuration

Husky is configured using the .husky directory at the root of the project. The pre-commit hook is configured to run lint-staged, which runs formatters and linters on staged files.

Usage

// Install Husky
npm run prepare

// Commit changes (Husky will run pre-commit hooks)
git commit -m "Your commit message"

lint-staged

Tool

A tool that runs linters and formatters on staged files. lint-staged is used in the Date Night App project to ensure that only staged files are formatted and linted before commits.

Configuration

lint-staged is configured using the .lintstagedrc.json file at the root of the project. The configuration specifies which commands to run on which files.

Usage

// Run lint-staged manually
npx lint-staged

ESLint

Tool

A static code analysis tool for identifying problematic patterns in JavaScript code. ESLint is used in the Date Night App project to enforce coding standards and catch potential errors.

Configuration

ESLint is configured using .eslintrc.json files in the server and client-angular directories. The configuration specifies rules for code quality, style, and best practices.

Usage

// Check for linting issues
npm run lint

// Fix linting issues
npm run lint:fix
