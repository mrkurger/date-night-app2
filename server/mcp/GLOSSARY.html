MCP Servers - Glossary

MCP (Model Context Protocol)

Architecture

A standardized protocol for interacting with different parts of the application ecosystem, providing consistent interfaces for various services and subsystems.

Usage: The MCP architecture allows for modular development and integration of specialized services.

MongoDB MCP Server

Database

An MCP server that provides optimized access to MongoDB database operations, including query optimization, data validation, and connection management.

Usage: import { mongodbMcp } from './mcp/mongodb/mongodb.mcp.js';

GitHub Actions MCP Server

CI/CD

An MCP server that interfaces with GitHub Actions for continuous integration and delivery, providing automation for testing, documentation generation, and deployment.

Usage: import { githubActionsMcp } from './mcp/github-actions/github-actions.mcp.js';

Terminal-Control MCP Server

Development

An MCP server that provides task automation for development workflows, build process management, and file operations handling with security constraints.

Usage: import { terminalControlMcp } from './mcp/terminal-control/terminal-control.mcp.js';

Mobile MCP Server

Mobile

An MCP server that supports mobile application development with testing infrastructure, responsive design validation, and app scraping capabilities.

Usage: import { mobileMcp } from './mcp/mobile/mobile.mcp.js';

MCP Controller

Code Pattern

A component that handles HTTP requests to MCP servers, processes inputs, invokes appropriate MCP service methods, and formats responses.

Usage: Used in route definitions to handle API endpoints for MCP servers.

MCP Service

Code Pattern

The core implementation of an MCP server that contains the business logic and operations for a specific domain.

Usage: Invoked by MCP controllers to perform domain-specific operations.

MCP Configuration

Code Pattern

Environment-specific settings and setup for an MCP server, including connection details, authentication, and operational parameters.

Usage: Used by MCP services to configure their behavior based on the environment.
