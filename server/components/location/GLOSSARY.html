Glossary: Location Component

Schemas

locationBaseSchema

Base Zod schema for validating location data. Defines the core structure and validation rules for location objects.

  • city: Required string
  • county: Required string
  • country: Optional string, defaults to "Norway"
  • coordinates: Required tuple of [longitude, latitude]
  • source: Optional enum of location data sources
  • population: Optional number
  • timezone: Optional string
  • postalCodes: Optional array of strings

searchQuerySchema

Zod schema for validating location search queries.

  • query: Required string for text search
  • limit: Optional number between 1-100, defaults to 10

nearbyQuerySchema

Zod schema for validating nearby location queries.

  • longitude: Required number between -180 and 180
  • latitude: Required number between -90 and 90
  • maxDistance: Optional number in meters, defaults to 10000
  • limit: Optional number between 1-100, defaults to 10

Validation Functions

LocationValidation

Object containing functions for validating different types of location-related data.

  • searchQuery: Validates search parameters
  • nearbyQuery: Validates nearby location parameters
  • idParam: Validates ID parameters
  • createLocation: Validates location creation data
  • updateLocation: Validates location update data

LocationValidator

Express middleware functions that use Zod schemas for request validation.

  • validateSearchQuery: Middleware for search endpoint
  • validateNearbyQuery: Middleware for nearby endpoint
  • validateIdParam: Middleware for ID validation
  • validateCreateLocation: Middleware for location creation
  • validateUpdateLocation: Middleware for location updates
