AI Lessons: Location Component

Migration Patterns

Express-Validator to Zod Migration

Key lessons learned when migrating from express-validator to Zod validation:

  • Create base schemas that can be reused and extended
  • Use Zod's type inference for better TypeScript integration
  • Leverage Zod's composability for complex validation logic
  • Implement proper error handling with descriptive messages

Validation Organization

Best practices for organizing validation code:

  • Separate schemas from middleware implementation
  • Group related schemas together
  • Make schemas as reusable as possible
  • Add clear documentation for each schema

Dependencies

Required Packages

  • zod: ^3.0.0 - Schema validation library
  • zod-validation-error: ^1.0.0 - Better Zod error messages

Testing Insights

Validation Testing

Important aspects to test in validation:

  • Valid input passes validation
  • Invalid input fails with correct error messages
  • Default values are correctly applied
  • Edge cases are properly handled

Error Cases

Common validation error scenarios to handle:

  • Missing required fields
  • Invalid coordinate values
  • Out of range values
  • Type mismatches
  • Malformed data structures

Performance Considerations

Validation Optimization

  • Cache compiled schemas when possible
  • Use appropriate error handling strategies
  • Consider validation order for better performance

Security Considerations

Input Sanitization

  • Always trim string inputs
  • Validate coordinate bounds
  • Prevent injection attacks
  • Implement proper access controls
