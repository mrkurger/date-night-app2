Changelog: Travel

This file tracks changes to the Travel component/module.

Unreleased

Added

  • Added comprehensive Zod validation schemas for travel operations:
    • Itinerary validation with nested object support
    • Location update validation with coordinate bounds
    • Query parameter validation for search and filters
    • Parameter validation for route URLs
  • Added type-safe validation middleware integration
  • Added runtime type checking for coordinates and dates
  • Added validation examples to documentation

Changed

  • Migrated from express-validator to Zod for type-safe validation
  • Updated all route handlers to use new validation system
  • Improved error messages with more specific validation feedback
  • Enhanced date validation with custom logic for arrival/departure times

Security

  • Enhanced input validation across all endpoints
  • Added strict type checking for geographical coordinates
  • Improved validation for dates and time ranges
  • Added parameter sanitization for MongoDB queries
