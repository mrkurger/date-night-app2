3. Using the Migration Tools

3.1 Creating Folder Documentation

To create the standard documentation files for a code folder:

./migrate-docs.sh create-folder-docs <folder-path>

Example:

./migrate-docs.sh create-folder-docs client-angular/src/app/features/chat

3.2 Converting Markdown to HTML

To convert a Markdown file to HTML:

./migrate-docs.sh convert-md-to-html <markdown-file> <output-html-file>

Example:

./migrate-docs.sh convert-md-to-html docs/features/CHAT.MD client-angular/src/app/features/chat/chat-system.html

3.3 Updating the Migration Checklist

To update the migration checklist with a new file mapping:

./migrate-docs.sh update-checklist <markdown-file> <html-file> <status>

Example:

./migrate-docs.sh update-checklist docs/features/CHAT.MD client-angular/src/app/features/chat/chat-system.html Completed

3.4 Auditing Documentation

To audit the current state of documentation:

./migrate-docs.sh audit-docs

4. Best Practices for Migration

  • Always update the CHANGELOG.html file in the target folder after migration
  • Add migration notes to the AILESSONS.html file
  • Ensure all links between documents are updated to point to the new HTML files
  • Add tooltips and links for function references where appropriate
  • Follow the component-library HTML template structure
  • Update the migration checklist after each file is migrated
  • Test all links and navigation after migration
