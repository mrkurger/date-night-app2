Overview

This glossary provides definitions for functions, scripts, and concepts used in the Angular client fix scripts. Use the alphabetical navigation below to quickly find terms.

A C E F N S

A

Angular Standalone Components Concept

A feature in Angular that allows components to be used without declaring them in an NgModule. Standalone components include their own imports array that specifies all the dependencies they need.

Example: 

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [
    CommonModule,
    NbCardModule,
    NbButtonModule
  ],
  template: `...`
})

C

Content Security Policy (CSP) Concept

A security standard that helps prevent cross-site scripting (XSS) and other code injection attacks. It specifies which resources (scripts, styles, fonts, etc.) are allowed to be loaded by the browser.

Example: 

<meta http-equiv="Content-Security-Policy" content="default-src 'self'; 
  style-src 'self' 'unsafe-inline' https://unpkg.com;">

ChatMessage Interface Concept

An interface that defines the structure of chat message objects in the application. It includes properties like sender, content, timestamp, and encryption data.

Example: 

export interface ChatMessage {
  id: string;
  roomId: string;
  sender: string | ChatParticipant;
  content: string;
  timestamp: Date;
  // Other properties...
}

E

Eva Icons Concept

A library of open-source icons used by Nebular and many Angular applications. It provides a set of consistent, customizable icons that can be used in the UI.

Example: 

<nb-icon icon="home-outline"></nb-icon>

F

fix-all-issues.cjs Script

The main script that runs all the fix scripts in sequence. It orchestrates the execution of individual fix scripts to address all issues in the Angular client.

Usage: 

node scripts/fix-all-issues.cjs

fix-chat-message-interface.cjs Script

A script that updates the ChatMessage interface in chat.service.ts to include all required properties used in components.

Usage: 

node scripts/fix-chat-message-interface.cjs

fix-css-paths.cjs Script

A script that creates placeholder CSS files for eva-icons and Nebular themes to prevent build errors when these files are referenced but not found.

Usage: 

node scripts/fix-css-paths.cjs

fix-index-html.cjs Script

A script that updates the index.html file to include Eva Icons CSS and proper Content Security Policy.

Usage: 

node scripts/fix-index-html.cjs

fix-nebular-imports.cjs Script

A script that consolidates multiple Nebular imports in component files into a single import statement.

Usage: 

node scripts/fix-nebular-imports.cjs

fix-remaining-nebular-issues.cjs Script

A script that fixes formatting and missing imports in components that use Nebular.

Usage: 

node scripts/fix-remaining-nebular-issues.cjs

fix-styles.cjs Script

A script that fixes styles.scss issues and adds CSS variables for Nebular theme.

Usage: 

node scripts/fix-styles.cjs

findTsFiles Function

A function used in several scripts to recursively find all TypeScript files in a directory.

Example: 

function findTsFiles(dir, fileList = []) {
  const files = fs.readdirSync(dir);
  
  files.forEach(file => {
    const filePath = path.join(dir, file);
    const stat = fs.statSync(filePath);
    
    if (stat.isDirectory()) {
      findTsFiles(filePath, fileList);
    } else if (file.endsWith('.ts') && !file.endsWith('.spec.ts')) {
      fileList.push(filePath);
    }
  });
  
  return fileList;
}

N

Nebular Concept

A UI library for Angular applications that provides a set of reusable components, themes, and utilities. It includes components like cards, buttons, forms, and more.

Example: 

<nb-card>
  <nb-card-header>Card Title</nb-card-header>
  <nb-card-body>Card Content</nb-card-body>
</nb-card>

NbDialogRef Concept

A reference to a dialog opened by Nebular's dialog service. It provides methods to close the dialog and retrieve data from it.

Example: 

constructor(public dialogRef: NbDialogRef<MyDialogComponent>) {}

closeDialog() {
  this.dialogRef.close(result);
}

NbTagComponent Concept

A Nebular component used to display tags or labels. It can be used with NbTagList and NbTagInput for tag input functionality.

Example: 

<nb-tag-list>
  <nb-tag *ngFor="let tag of tags" [text]="tag" removable></nb-tag>
  <input nbTagInput [separatorKeys]="separatorKeysCodes">
</nb-tag-list>

S

separatorKeysCodes Concept

An array of key codes used by NbTagInput to determine which keys should trigger the creation of a new tag. Common values include Enter (13) and comma (188).

Example: 

readonly separatorKeysCodes: number[] = [13, 188]; // Enter and comma

U

update-primeng-imports.js Script

A script that analyzes and optimizes PrimeNG module imports in Angular components. It consolidates imports, adds missing modules based on template usage, and ensures proper module inclusion in standalone components.

Example: 

node update-primeng-imports.js src/app/components

M

migrate-to-primeng.js Script

A script that migrates Nebular component selectors and class names to their PrimeNG equivalents. The script processes TypeScript, HTML, and SCSS files to replace Nebular references with PrimeNG.

Example: 

node migrate-to-primeng.js src/app/components

P

PrimengButtonComponent Component

A wrapper component for PrimeNG Button with additional features such as tooltips and standardized styling. Uses standalone component pattern with proper module imports.

Example: 

@Component({
  selector: 'app-primeng-button',
  standalone: true,
  imports: [CommonModule, ButtonModule, TooltipModule],
  template: `
    
    
  `
})
