Security Enhancement - Current Date

Enhanced URL validation to prevent XSS attacks

  • Expanded the check for dangerous URL protocols beyond just javascript:
  • Added checks for additional potentially dangerous protocols including data:, vbscript:, file:, about:, blob:, and others
  • Improved code organization by using an array of dangerous protocols for better maintainability

This change significantly improves security by preventing various protocol-based XSS attack vectors that could be exploited through URL inputs.

Validation Utils Changelog

2025-04-11

April 11, 2025

FIX: Replaced global isNaN() with more robust Number.isNaN() in the isValidISODate() function. This change improves type safety and prevents unexpected behavior when validating date strings.

File: validation.utils.ts

Reason: The global isNaN() function attempts to convert its argument to a number before checking, which can lead to unexpected results with non-numeric inputs. Number.isNaN() is more strict and only returns true if the value is exactly the NaN value, providing more predictable behavior.
