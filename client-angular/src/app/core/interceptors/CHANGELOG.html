2024-05-01: Security Fixes in HTTP Error Interceptor

May 1, 2024

Security Fixed multiple security vulnerabilities

  • Fixed Server-Side Request Forgery (SSRF) vulnerability in shouldSkipErrorHandling method by properly validating and sanitizing URLs before processing them.
  • Fixed Prototype Pollution vulnerability in sanitizeObject method by using Object.getOwnPropertyNames instead of Object.keys to avoid traversing the prototype chain.
  • Fixed Duplicate Object Property issue in trackError method by removing the duplicate errorCode property.
  • Updated trackError method to properly match the ErrorTelemetry interface, fixing TypeScript errors.

Implementation Details

  • Added URL validation in shouldSkipErrorHandling to only allow HTTP and HTTPS protocols.
  • Improved URL path comparison to use exact matching instead of substring matching.
  • Added path normalization to prevent path traversal attacks.
  • Updated error tracking to use the correct property names according to the ErrorTelemetry interface.
  • Enhanced object sanitization to prevent prototype chain traversal using Object.getOwnPropertyNames.
  • Created objects with Object.create(null) to avoid prototype pollution in sanitization methods.
  • Added type checking for URL parameters to prevent type confusion attacks.
