Shared Module Changelog

2025-05-16: Fixed Component ID Collision Issues

May 16, 2025
Fixed: Component ID generation collision detected errors (NG0912) for Nebular components
Added: Custom component ID generator to avoid collisions between Nebular components
Changed: Updated NebularModule to use the custom component ID generator
Fixed: Service worker registration failure (NG05604) by disabling it in development mode
Added: Placeholder ngsw-worker.js file to prevent 404 errors during development

Technical Details

The application was experiencing multiple "Component ID generation collision detected" errors (NG0912) for Nebular components. This was fixed by implementing a custom component ID generator that adds a unique prefix and random string to each component ID.

The service worker registration was failing with a 404 error because the ngsw-worker.js file was missing. This was fixed by creating a placeholder file and disabling the service worker in development mode.

Code changes: 


// Custom component ID generator to avoid collisions
export function customComponentIdGenerator(componentType: any): string {
  // Add a prefix to the component ID to make it unique
  return `custom-${componentType.name}-${Math.random().toString(36).substring(2, 8)}`;
}

// Provider in NebularModule
providers: [
  // Provide the custom component ID generator
  {
    provide: 'componentIdGenerator',
    useValue: customComponentIdGenerator,
  },
],
