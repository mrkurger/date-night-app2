Glossary: Icon

This file contains documentation for the Icon component system and its related services.

IconComponent

Component | @Component
A reusable component for displaying icons using PrimeIcons. Provides consistent styling and behavior across the application.

Properties

Name Type Description
@Input() name string The name of the icon to display
@Input() filled boolean Whether to use filled variant (default: true)
@Input() size number Icon size in pixels (default: 24)
@Input() color string Optional color name from design system

Usage Example

<app-icon
  name="heart"
  [filled]="true"
  [size]="32"
  color="primary"
></app-icon>

IconService

Service | @Injectable
Service that manages icon name conversions and mappings between different icon libraries. Provides a centralized way to convert legacy icon names to PrimeIcons.

Methods

Name Parameters Returns Description
getIconName name: string
filled?: boolean 		string Converts any icon name to its PrimeIcons equivalent

Usage Example

// In a component
constructor(private iconService: IconService) {}

ngOnInit() {
  const iconClass = this.iconService.getIconName('fa-heart');
  // Returns: "pi pi-heart"
}

Icon Name Mapping

Functionality
The icon system provides automatic mapping between different icon libraries:
  • Font Awesome (fa-*) → PrimeIcons (pi-*)
  • Material Icons → PrimeIcons
  • Eva Icons → PrimeIcons

Common Icon Mappings

Legacy Name PrimeIcon Name
fa-home, home pi pi-home
fa-heart, heart pi pi-heart
fa-search, search pi pi-search
fa-user, user pi pi-user
fa-cog, settings pi pi-cog
