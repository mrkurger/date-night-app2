Migration Strategy

  1. Parallel Integration:
    • Created PrimeNGModule alongside NebularModule
    • Both UI libraries available during migration
    • Components migrated one at a time
  2. Migration Order:
    • Start with simpler, self-contained components
    • Progress to more complex, interconnected components
    • Navigation and layout components last
  3. Testing Strategy:
    • Unit tests updated alongside component migration
    • Visual regression testing for each component
    • End-to-end tests maintained throughout

Theme Migration

PrimeNG themes will be customized to match our current Nebular theme. Key color variables and design tokens will be mapped across.

Note: PrimeNG uses a different theming system. Colors and styles will be mapped through CSS variables to maintain consistency.
