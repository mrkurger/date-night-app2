AI Lessons: Advertiser browsing alternate

This file contains lessons learned and best practices for the Advertiser browsing alternate component/module.

Component Architecture

Layout and Theming

Key lessons from migrating to PrimeNG:

  • PrimeNG's theme system is highly adaptable and can match most Nebular color schemes
  • CSS custom properties enable smooth theme transitions and maintenance
  • Component decomposition improves maintainability and reusability
  • PrimeNG's layout primitives (Panel, Toolbar) provide consistent structure

State Management

  • Keep state management close to where it's used
  • Use clear parent-child component hierarchies
  • Leverage PrimeNG's built-in state management for menus and tabs

Dependencies

PrimeNG Modules

  • CardModule - Content containers and layouts
  • ButtonModule - Interactive elements
  • ToolbarModule - Navigation and action bars
  • InputTextModule - Form inputs
  • TabViewModule - View switching
  • MenuModule and TieredMenuModule - Navigation menus
  • TooltipModule - User guidance
  • PanelModule - Layout structure

Performance Optimizations

  • Theme switching is optimized using CSS custom properties
  • Lazy loading child components improves initial load
  • PrimeNG provides efficient event handling

Accessibility

  • PrimeNG components include built-in ARIA support
  • Theme variables maintain WCAG color contrast
  • Keyboard navigation is properly handled

Testing Strategy

  • Mock PrimeNG services in unit tests
  • Test theme switching functionality
  • Verify component interactions
  • Check responsive behavior across breakpoints

Documentation Best Practices

  • Keep component documentation in sync with changes
  • Document all theme customizations
  • Maintain clear interface definitions
  • Include accessibility considerations
