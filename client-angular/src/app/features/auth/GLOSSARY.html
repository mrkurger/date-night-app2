Glossary: Auth

This file contains a glossary of functions and methods defined in the Auth component/module.

This file is automatically generated. Do not edit it directly.

A component that handles OAuth authentication callbacks from external providers. It processes authentication results and redirects users to appropriate destinations based on success or failure of the authentication attempt.

This component is designed to work with multiple OAuth providers (Google, Facebook, Apple, Microsoft) and determines the provider from query parameters.

AuthLayoutComponent

Type: Component | Path: /features/auth/components/auth-layout/auth-layout.component.ts

A container component that provides consistent layout and styling for all authentication-related pages. It includes the application branding and a router outlet for rendering nested authentication components like login, register, and password reset.

LoginComponent

Type: Component | Path: /features/auth/login/login.component.ts

Provides a form for user authentication. Supports traditional email/password login as well as social login options through OAuth providers. Handles form validation, submission, and error feedback.

Methods

  • onSubmit()Handles form submission for traditional login. Validates the form and attempts to authenticate the user with the provided credentials.
  • socialLogin(provider: string)Initiates OAuth authentication flow with the specified provider (e.g., 'google', 'facebook', 'apple', 'microsoft').

RegisterComponent

Type: Component | Path: /features/auth/register/register.component.ts

Provides a form for new user registration. Supports both traditional registration with email/password and social registration through OAuth providers. Handles form validation, submission, and error feedback.

Methods

  • onSubmit()Handles form submission for traditional registration. Validates the form and attempts to register a new user with the provided information.
  • socialLogin(provider: string)Initiates OAuth registration flow with the specified provider (e.g., 'google', 'facebook', 'apple', 'microsoft').
  • onCancel()Redirects the user back to the login page.

RequestPasswordComponent

Type: Component | Path: /features/auth/components/request-password/request-password.component.ts

Provides a form for users to request a password reset. Users enter their email address, and if valid, a password reset link is sent to that address.

Methods

  • onSubmit()Handles form submission for password reset requests. Validates the email and sends a reset link if valid.

ResetPasswordComponent

Type: Component | Path: /features/auth/components/reset-password/reset-password.component.ts

Allows users to set a new password after receiving a password reset link. The component validates the reset token and provides a form for entering a new password.

Methods

  • onSubmit()Handles form submission for password reset. Validates the form and attempts to update the user's password with the provided new password.

AuthService.requestPassword

Type: Method | Path: /core/services/auth.service.ts

Sends a request to the server to initiate the password reset process for a user's account.

Parameters

Name Type Description
email string The email address of the user's account

Returns

Observable<void> - An observable that completes when the request is processed

AuthService.resetPassword

Type: Method | Path: /core/services/auth.service.ts

Completes the password reset process by sending the reset token and new password to the server.

Parameters

Name Type Description
token string The reset token from the password reset link
password string The new password for the user's account

Returns

Observable<void> - An observable that completes when the password is reset
